## Overview

The Club Las Aves registration form implements dynamic DOM manipulation to allow users to add and remove sports from the selection list. This feature demonstrates practical use of JavaScript's DOM API without relying on frameworks.

## Dynamic Sports System

### Data Structure

The system maintains a list of additional sports and tracks which ones have been added:

```javascript app.js
let deportes = ["Golf", "Boxeo", "Bádminton", "Frontón", "Hockey"];
let deportesAñadidos = []; // Array para rastrear qué deportes se añadieron
```

- **deportes**: Array of 5 additional sports that can be added
- **deportesAñadidos**: Tracks added sports and their DOM elements for removal

### HTML Structure

The sports are contained in a parent container:

```html index.html
<div class="listDeportes" id="listDeportes">
    <label>
        <input type="checkbox" name="deportes" value="natacion">
        Natación
    </label>
    <label>

        Fútbol

    <label>
        <input type="checkbox" name="deportes" value="padel">
        Pádel
    </label>

        <input type="checkbox" name="deportes" value="tenis">

</div>

<button type="button" id="añadirBOM">Añadir deporte</button>

```

## Adding Sports Dynamically

### Implementation

The add button dynamically creates new checkbox elements:

```javascript app.js
// Añadir deporte
document.getElementById("añadirBOM").addEventListener("click", function (e) {
    e.preventDefault();
    var msgMods = document.getElementById('msgMods');

    if (deportesAñadidos.length < deportes.length) {
        let lista = document.getElementById("listDeportes");
        let deporte = deportes[deportesAñadidos.length];

        let label = document.createElement("label");
        let input = document.createElement("input");

        input.type = "checkbox";
        input.className = "deporte añadido";
        input.name = "deportes";
        input.value = deporte.toLowerCase();

        input.addEventListener('change', validarCheckboxes);

        label.appendChild(input);
        label.appendChild(document.createTextNode(" " + deporte));
        lista.appendChild(label);

        deportesAñadidos.push({
            deporte: deporte,
            label: label
        });
        validarCheckboxes();
    } else {
        msgMods.style.color = '#d62323';
        msgMods.textContent = 'Llegaste al límite, no hay más deportes :(';
    }
});
```

## Removing Sports Dynamically

### Implementation

The remove button deletes the most recently added sport:

```javascript app.js
// Eliminar deporte (AHORA SÍ ELIMINA UNO POR UNO)
document.getElementById("eliminarBOM").addEventListener("click", function (e) {
    e.preventDefault();
    const msgMods = document.getElementById('msgMods');

    if (deportesAñadidos.length > 0) {
        let ultimo = deportesAñadidos.pop();
        ultimo.label.remove(); // Elimina el label y el checkbox dentro

        // Volver a contar después de eliminar
        validarCheckboxes();
    }
});
```

## Key DOM Methods Used

### Creating Elements

<Accordion title="document.createElement()">
Creates a new HTML element:

```javascript
let label = document.createElement("label");
let input = document.createElement("input");
```
</Accordion>

<Accordion title="document.createTextNode()">
Creates a text node for inserting text content:

```javascript
label.appendChild(document.createTextNode(" " + deporte));
```

This is safer than using `innerHTML` and prevents XSS vulnerabilities.
</Accordion>

### Modifying Elements

<Accordion title="element.appendChild()">
Adds a child node to an element:

```javascript
label.appendChild(input);
lista.appendChild(label);
```
</Accordion>

<Accordion title="element.remove()">
Removes an element from the DOM:

```javascript
ultimo.label.remove();
```

This is a modern method that's cleaner than `parentNode.removeChild()`.
</Accordion>

### Selecting Elements

<Accordion title="document.getElementById()">
Selects a single element by ID:

```javascript
let lista = document.getElementById("listDeportes");
```
</Accordion>

<Accordion title="document.querySelectorAll()">
Selects all matching elements (used in validation):

```javascript
var checkboxes = document.querySelectorAll('input[name="deportes"]:checked');
```
</Accordion>

## Event Handling

### Event Delegation

The validation listener is attached to the parent container, not individual checkboxes:

```javascript app.js
document.getElementById('listDeportes').addEventListener('change', validarCheckboxes);
```

**Benefits:**
- Works for dynamically added checkboxes automatically
- Only one event listener instead of many
- Better performance
- No need to attach listeners to new elements

### Dynamic Event Listeners

Each dynamically created checkbox still gets its own listener for immediate feedback:

```javascript
input.addEventListener('change', validarCheckboxes);
```

This ensures the validation function runs when the checkbox is toggled.

## State Management

### Tracking Added Sports

The `deportesAñadidos` array stores objects with two properties:

```javascript
{
    deporte: "Golf",        // Sport name as string
    label: labelElement     // DOM element reference
}
```

This dual storage allows:
- **Displaying the sport name** in messages
- **Removing the exact DOM element** when the user clicks remove

### Array Operations

## Limit Enforcement

### Maximum Sports

The system prevents adding more than 5 additional sports:

```javascript
if (deportesAñadidos.length < deportes.length) {
    // Add sport
} else {
    msgMods.style.color = '#d62323';
    msgMods.textContent = 'Llegaste al límite, no hay más deportes :(';
}
```

**Total possible sports:**
- 4 default sports (Natación, Fútbol, Pádel, Tenis)
- 5 additional sports (Golf, Boxeo, Bádminton, Frontón, Hockey)
- **Maximum: 9 sports total**

However, validation limits selection to only 1-3 sports.

## Integration with Validation

### Automatic Revalidation

After every add/remove operation, the validation function is called:

```javascript
validarCheckboxes();
```

This ensures:
- The selection count message updates immediately
- Users see current selection limits
- Form submission validation remains accurate

<Note>
**Best Practice**: Always revalidate after DOM mutations that affect form state.
</Note>

## Performance Considerations

<Tip>
**Efficient DOM Operations**: Creating elements programmatically with `createElement()` is more efficient than parsing HTML strings with `innerHTML`.
</Tip>

<Warning>
**Memory Management**: The `deportesAñadidos` array maintains references to DOM elements. These are automatically garbage collected when removed from the array.
</Warning>

## Browser Compatibility

- `createTextNode()` - All browsers
- `appendChild()` - All browsers
- `remove()` - IE11+ (polyfill available)
- `addEventListener()` - All modern browsers
- `querySelectorAll()` - IE9+
